Lifecycle flow how a CD becomes a CDT

1
CIP / KYC at CU Off-chain membership verification (not in this UI)
2
Issue credentials NCUA → institution VC; CU → AccountHolder VC
3
Fund CD Deposit into cd_funding account in bank core
4
Oracle verifies + attests VC presentation + product min → signed payload
5
Mint CDT Oracle co-signs; vault locks principal + interest
6
Redeem / early exit Burn CDT; payout per vault rules

Steps 1–4 (bank + oracle) are live against Docker Postgres below. Steps 5–6 ran successfully in the in-process Lucid emulator demo (offchain/demo) — not a public Cardano node in this view.

Credential trust chain DID solution

NCUA (trusted root DID)
└─ issues InsuredInstitutionCredential
→ Credit Union DID
└─ issues AccountHolderCredential
→ Member DID (holder)
Member presents both → oracle verifyPresentation
No valid chain ⇒ no attestation ⇒ no mint
